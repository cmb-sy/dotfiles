---
name: reflect-review
argument-hint: "[--weekly | --monthly]"
---

reflect_log の累積データを分析し、成長を言語化する。さらに正答率が低い問題を再出題して定着を確認する。

**モード:**
- **`--weekly`（デフォルト）**: 直近7日間のセッションを対象
- **`--monthly`**: 直近30日間のセッションを対象

---

## 処理フロー

### Step 1: データ読み込み

`01_quant/reflect_log.md` を読み込む。

- ファイルが存在しない場合: 「reflect_log.md が見つかりません。先に /reflect を実行してください」と通知して終了
- サマリー表とセッション詳細の両方を読み込む
- 対象期間（weekly = 直近7日 / monthly = 直近30日）でフィルタリングする
- 対象セッションが0件の場合: 「対象期間にセッションがありません」と通知して終了

---

### Step 2: 成長分析

対象期間のデータから以下を算出する。

**数値分析:**

| 指標 | 算出方法 |
|---|---|
| 総セッション数 | 対象期間のセッション数 |
| 総問題数 | 全問の合計 |
| 総正答率 | 正解数 / 総問題数 |
| 軸別正答率 | 技術的理解 / 判断根拠 / 意義北極星 / リスク それぞれの正解率 |
| セッション別推移 | 日付順に正答率を並べる |
| 最も弱い軸 | 軸別正答率が最低のもの |
| 改善が見られる軸 | 直近セッションで正答率が上がったもの |

**弱点問題の抽出:**

- 全セッションから「x（不正解・わからない）」の問題を抽出する
- 同じ軸・同じ要旨の問題が複数回 x なら優先順位を上げる
- 上位3〜5問を「再出題リスト」として選定する

---

### Step 3: 成長の言語化

分析結果をもとに成長レポートを生成して表示する。

**表示フォーマット（プレーンテキストのみ。特殊文字・絵文字禁止）:**

```
## reflect-review レポート（{期間} / {YYYY-MM-DD}）

対象: {N}セッション / {M}問

### 正答率推移

{日付} セッション: {N}/{M}問 ({X}%)
{日付} セッション: {N}/{M}問 ({X}%)
...

全体: {N}/{M}問 ({X}%)

### 軸別正答率

技術的理解:   {X}% ({N}/{M}問)
判断根拠:     {X}% ({N}/{M}問)
意義北極星:   {X}% ({N}/{M}問)
リスク:       {X}% ({N}/{M}問)

### 成長の言語化

{3〜5行で以下を自然な文章で記述:}
- 全体的な正答率の傾向（伸びているか、停滞しているか）
- 得意になってきた軸とその根拠
- まだ弱い軸とその傾向
- Claude Code との作業を通じて何が身についてきているか
- 北極星（フリーランス独立）との接続で見た成長の実感

### 再出題リスト（弱点 {N}問）

1. [{軸}] {問題の要旨} — {x が何回あったか}
2. ...
```

---

### Step 4: 弱点再出題

Step 2 で選定した問題を AskUserQuestion で1問ずつ出題する。

- 問題・選択肢は reflect_log の解説まとめから再構成する
- 選択肢に「わからない」を必ず含める
- 前回の回答（x の内容）は提示しない（先入観を与えないため）
- 各問の回答後に 用語解説 / 解説 / Tips を表示する（Step 3 の形式と同じ）

**回答結果の記録:**

- 正解した場合: reflect_log の該当行の正否列に「o（再正解）」を追記する
- 再度不正解の場合: 「o（再正解）」は付けない

---

### Step 5: レポート書き込み

`01_quant/reflect_log.md` の末尾にレビューセクションを追記する。

```markdown
---

## review {YYYY-MM-DD} | {weekly / monthly} | 全体 {X}%

| 軸 | 正答率 | 傾向 |
|---|---|---|
| 技術的理解 | {X}% | 改善 / 停滞 / 悪化 |
| 判断根拠 | {X}% | ... |
| 意義北極星 | {X}% | ... |
| リスク | {X}% | ... |

### 成長メモ
{Step 3 の成長の言語化テキスト}

### 再出題結果
| 問題の要旨 | 前回 | 今回 |
|---|---|---|
| {問題} | x | o / x |
```

---

## ルール

1. reflect_log.md が存在しない・対象期間にデータがない場合は処理を中止する
2. 成長の言語化は数値の羅列ではなく「何ができるようになったか」を主語にして書く
3. 弱点再出題は「責めない」—— ギャップの可視化と定着が目的
4. 表示はプレーンテキストのみ。特殊文字・絵文字・装飾記号は使わない
5. 再出題の問題は reflect_log の解説から再構成し、新たに WebSearch はしない（既出問題の定着確認のみ）
6. サマリー表の北極星接続度・弱点軸の列は最新セッション基準で更新しない（reflect が担当するため）
